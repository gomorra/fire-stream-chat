# 2026-04-11/12 Sprint Test Plan

This plan automates ~35 of the 40 test cases that landed across PRs #16–#23
on 2026-04-11/12. The remaining ~5 are documented as manual tests because
they require a real device, Firebase backend, or windowing system that the
JVM test runtime cannot fake.

## Coverage summary

| Category | Count | Tooling |
|---|---|---|
| Pure unit (MockK / fakes) | 28 | `./gradlew :app:testDebugUnitTest` |
| Compose UI (Robolectric + createComposeRule) | 10 | same |
| Screenshot baselines (Roborazzi) | 4 | `recordRoborazziDebug` / `verifyRoborazziDebug` |
| Manual smoke | 5 | manual on device |

All automated tests run as part of the standard `./gradlew test` invocation.
The Roborazzi `verify*` tasks run automatically after `testDebugUnitTest`.

## Manual tests (5 remaining)

These five test cases require infrastructure the JVM test runtime cannot fake.
Run them once on a real device per release.

### TC-15 — End-to-end 1:1 message delivery

**Why manual:** Requires two authenticated Firebase users + real Firestore.

1. Sign in as user A on device 1, user B on device 2.
2. From A, send a text message to B.
3. Verify on device 2:
   - Message arrives within 5 s.
   - Push notification posts unless B already has the chat open.
   - Read receipt turns blue on A within 2 s of B opening the chat.
4. From B, reply with an image attachment.
5. Verify on A: image renders at correct aspect ratio, upload progress overlay
   disappears at 100 %, downloaded copy survives an app cold restart.

### TC-16 — End-to-end group message delivery

**Why manual:** Same as TC-15, plus unread fan-out across N participants.

1. Create a group with 3+ users.
2. From any participant, send a text message and an image.
3. Verify each non-sender's chat-list badge increments by 1.
4. Verify the Cloud Function `sendPushNotification` posts to all non-sender FCM tokens.
5. Open the chat on each non-sender device — badge should drop to 0 within 2 s.

### TC-22 — Link preview WebView screenshot rendering

**Why manual:** Requires a real windowing system; PixelCopy needs a real Window.

1. Send a message containing `https://github.com` (or any URL whose page has
   no `og:image`/`twitter:image`/`apple-touch-icon`).
2. Verify the link-preview card appears within ~3 s with a screenshot of the
   page header (not a blank image).
3. Tap the preview thumbnail → fullscreen viewer opens with the same image.

### TC-23 — Link preview WebView fallback for sites with no meta image

**Why manual:** Same as TC-22 — verifies the `WebPagePreviewCapture` Dialog +
PixelCopy fallback path from commits `2862445` → `1b85356` → `0d570f1`.

1. Cold-start the app, then send a message with a URL that has only a `<title>`
   tag and no image meta tags.
2. Verify a link preview appears with the title and a WebView screenshot.
3. Force-close and reopen the app, scroll to the message — the cached preview
   image should still be present.

### TC-37 — Profile screen camera/gallery picker

**Why manual:** Requires real `ActivityResultLauncher` + system camera intent.

1. Open Profile → tap the avatar → "Take photo".
2. Capture a photo, confirm it.
3. Verify the new avatar appears within 2 s and survives an app restart.
4. Repeat with "Pick from gallery".

### TC-38 — Group settings camera/gallery picker

**Why manual:** Same as TC-37, but on the group avatar update flow.

1. Open Group settings → tap the group avatar → choose camera or gallery.
2. Confirm selection.
3. Verify the new group avatar propagates to the chat list and to other
   participants' chat lists within 5 s (Firestore listener fan-out).

```bash
# Just the new sprint tests
./gradlew :app:testDebugUnitTest \
  --tests "com.firestream.chat.data.remote.fcm.ActiveChatTrackerTest" \
  --tests "com.firestream.chat.data.remote.LinkPreviewSourceTest" \
  --tests "com.firestream.chat.data.util.MediaFileManagerTest" \
  --tests "com.firestream.chat.ui.chat.ChatInfoManagerTest" \
  --tests "com.firestream.chat.ui.chat.ChatMessageLoaderUnreadTest" \
  --tests "com.firestream.chat.ui.chat.ChatViewModelBlockedStateTest" \
  --tests "com.firestream.chat.ui.chat.MessageBubbleSmokeTest" \
  --tests "com.firestream.chat.ui.chat.MessageBubbleScreenshotTest" \
  --tests "com.firestream.chat.ui.chatlist.ChatListItemUiTest"

./gradlew :app:verifyRoborazziDebug
```
